# Prova 01 Escobar

### Description

- Um Adendo, quando for testar voce deve entrar na pasta das questoes para realizar os testes pois os objetos nao estao com package por questao.

## Questao 01 - Strategy

### Justificativa: 
- Eu escolhi utilizar o modelo Strategy pois no exercicio pedia para que houvesse um tratamento especial para cada tipo de calculo financeiro, e cada um tem seu próprio tipo, o strategy basicamente é um pattern que trata diferentes tipos de regras de negocio utilizando uma unica interface, facilitando a vida do programador sem ele precisar saber o que acontece em cada tipo de classe, somente tendo que instaciar aquela que ele quiser usar no calculo.

## Questao 02 - Adapter

### Justificativa

- Eu escohli utilizar o modelo adapter justamente porque se faz necessário traduzir um sistema legado para um sistema atual, e o adapter é perfeito para isso, poios o padrao dele visa criar uma classe que faz a tradução de um sistema para outro, fazendo com que eu consiga utilizar funcionalidades dos softwares antigos sem precisar refatora-los. 

## Questao 03 - State

### Justificativa

- Eu escolhi utilizar o modelo state, pois no exercicio era pedido que a máquina trocasse de estado a cada momento em que ela estava ou aquecida ou em estado desligado, então a cada momento havia um estado, entao o state se encaixe aqui pois ele justamente trata com os varios estados de uma coisa só.

## Questao 04 - Chain of Resposibility

### Justificativa

- Eu escolhi usar o chain of resposibility, pois o exercicio pedia par aque fosse validado nfs, e basicamente para voce fazer uma boa validação seria passar por etapas, com o chain voce cria uma corrente de etapas para conseguir validar alguma coisa, por exemplo aqui a nf.
